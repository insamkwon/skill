### 🎯 이 스킬은 뭘까?

### 🚀 진입점별 추천 경로




상황 시작점 다음 단계
"완전히 새로운 스킬을 만들고 싶어" 새 스킬 생성 → 검증 → 배포
"기존 스킬이 잘 작동하는지 확인하고 싶어" 스킬 진단 → 개선 또는 유지
"전체 스킬 현황 파악하고 싶어" 일괄 진단 → 필요 시 개선
"스킬 두 개 비교하고 싶어" A/B 비교 → 선택 적용
"스킬 롤백 준비하고 실험하고 싶어" 스냅샷 & 롤백 → 실험 → 결정
"사용하지 않는 스킬 정리하고 싶어" 스킬 폐기 → 삭제

#### Step 2: plan.md 작성 (PLAN 단계)


📋 "무엇을, 왜, 어떻게 만들지 설계"

# Skill Plan: meeting-summary

## 목적 (Purpose)
회의 내용을 자동으로 정리하여 구조화된 문서로 생성

## 성공 기준 (Success Criteria)
- [ ] "회의록 정리해줘"에 반응
- [ ] 참석자, 안건, 결론, Action Item 추출
- [ ] 마크다운 형식으로 출력
- [ ] 3분 내에 완료

## 트리거 (Triggers)
- 명시적: "회의록 정리", "meeting summary", "회의 요약"
- 암묵적: 회의 관련 대화 시

## 경계 (Boundaries)
- 하는 것: 회의 내용 정리, Action Item 추출
- 안 하는 것: 회의 녹음 (→ 별도 도구), 음성-텍스트 변환 (→ 별도 도구)

## 위임 관계 (Delegates To)
| 상황 | 위임 대상 스킬 |
|------|---------------|
| 음성 녹음 필요 시 | 없음 (수동 처리) |
